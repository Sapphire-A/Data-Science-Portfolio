VIEWS, CONSTRAINTS, INDEXES

VIEWS

Views are temporary tables created from the basic tables to highlight specific designations or features or to show a broad range of features as required by the company or the analyst. They make analysis easier. Creating a view requires a lot miore specication or precision than creating a simply table and a lot more rules apply. For example the table headers or column titles must have unique names, as must the View itself. Furthermore, some clauses like the 'ORDER BY' clause cannot be used in the query unless the 'TOP' aggregate is used in the SELECT statement; and so on. a view can be a union of other views or tables as in the Roxy example below.


CONSTRAINTS

Constraints are checks set up in tables to preserve the integrity of the data, that is, to ensure that only data that meet the set specifications are allowed into the specified column. They can be created with the table initially, or added to an already existing table. The syntax used in both cases are slightly different.

CHECK constraints were put in the fields/columns representing BTA and PTA which have a limit of 5000 and 4000 respectively. They were equally put in the CBN Buy_Amount column as this Amount has a cap of $50000.

UNIQUE constraints were also put in various fields including all item ID's that are not Primary Keys


INDEXES

Indexes are a speedup mechanism to quickly find important or significant features of data within database tables or columns.
Indexes were placed in relevant tables and columns for quick reference such as the Customer table and the Forex Purchases tables to prioritise significant information.

    Create index idx_Ocean on Customer
    (Occupation, Acquaintance, nationality)
    Create index idx_TA On Forex_sales (BTA, PTA)
    Create index idx_fax on CBN_Purchases
    (FX_Rate, Amount_USD)

STORED PROC

Stored Procedures are SQL statements stored in the database that can be executed or called in queries. Using a stored procedure allows encapsulation of complicated or frequently used logic, and improves query performance by utilizing cached query plans. They can return any value a standard query can return. Other benefits over dynamic SQL expressions
